# 1 Введение

<a name="appointment"/>

## 1.1 Назначение
В этом документе описаны функциональные и нефункциональные требования к десктопному приложению «MFDictionary» для операционной системы Windows. Этот документ предназначен для команды, которая будет реализовывать и проверять корректность работы приложения. Необходимо разработать программу для создания словаря иностранных слов, которые выберет сам пользователь. 

<a name="business_requirements"/>

## 1.2 Бизнес-требования

<a name="initial_data"/>

### 1.2.1 Исходные данные
Знание иностранного языка всегда ценилось, но в наше время – это просто стало необходимостью. Без знания английского языка невозможно получить образование заграницей, найти престижную работу и даже просто попутешествовать по зарубежным странам. Изучать иностранный язык лучше с раннего детства, но, тем не менее, начать обучение можно в любом возрасте. Благодаря многочисленным приложениям повысить свой уровень знаний в какой-либо сфере в современном мире не составляет особого труда. Исключением не стало и изучение иностранных языков.

<a name="business_opportunities"/>

### 1.2.2 Возможности бизнеса
Многие люди любой возрастной категории желают найти приложение для своего компьютера, которое позволит наиболее продуктивно изучать иностранные слова. Подобная программа позволит им повысить их словарных запас с нескольких десятков до нескольких сотен и даже тысяч слов. Многим не хватает мотивации продолжать обучение, но возможность пройти тест по переводу добавленных в словарь слов только может стать стимулом для людей, уже использующих ресурс, а также позволит увеличить количество новых пользователей.

<a name="project_boundary"/>

### 1.2.3 Границы проекта
Приложение «MFDictionary» позволит пользователям найти интересуемое слово вместе с переводом на любой выбранный иностранный язык с примерами использования путём поиска, а также эффективно запоминать их путём прохождения теста. Ресурс предоставляет возможность просмотреть правильный ответ на те вопросы теста, на которые был дан неверный ответ. 

<a name="analogues"/>

## 1.3 Аналоги
На данный момент полностью аналогичного приложения не существует. С точки зрения перевода аналогами могут служить десктопные приложения-переводчики такие, как «Google.Translate» и «Яндекс.Переводчик». С точки зрения изучения языка и прохождения для этой цели тестов аналогом является Lingualeo. Однако данный сервис ограничен исключительно изучением английского языка, а также имеет только версии в виде веб-приложения и мобильного приложения.

# 2 Требования пользователя

<a name="software_interfaces"/>

## 2.1 Программные интерфейсы
Продукт должен являться десктопным приложением для операционной системы Windows версии 7 и выше, а также иметь user-friendly интерфейс. Приложение должно быть реализовано с использованием графической подсистемы Microsoft — WPF. Бизнес-логика приложения будет написана на языке C# с применением технологии .NET Framework, а графический интерфейс с помощью декларативного языка разметки — XAML. Изучаемые иностранные слова должны храниться в базе данных MS SQL. Поиск слов осуществляется благодаря использованию API «Яндекс.Словарь».

# 3 Системные требования

<a name="functional_requirements"/>

## 3.1 Функциональные требования

<a name="main_functions"/>

### 3.1.1 Основные функции

<a name="add_word_to_dictionary"/>

#### 3.1.1.1 Добавление слова в словарь
**Описание.** Пользователь имеет возможность создать свой личный словарь иностранных слов путем поиска нужных слов. В основе поиска лежит использование сервиса «Яндекс.Словарь».

| Функция | Требования | 
|:---|:---|
| Поиск слова | Приложение должно предоставить пользователю возможность найти слово с его переводом и примерами использования для последующего добавления в словарь |
| *Такое слово уже присутствует в словаре* | *Приложение должно известить пользователя о том, что слово уже есть в словаре и не будет добавлено в него повторно* |
| *Искомое слово не было найдено*| *Приложение должно известить пользователя об ошибке поиска, если слово не было найдено. Возникновение ошибки может быть обусловлено либо орфографической ошибкой в введенном слове, либо его отсутствием в базе данных сервиса «Яндекс.Словарь»* |

<a name="remove_word_from_dictionary"/>

#### 3.1.1.2 Удаление слова из словаря
**Описание.** Пользователь имеет возможность удалить слово из словаря.

**Требование.** Приложение должно предоставить пользователю возможность удалить слово из словаря.

<a name="passing_the_test"/>

#### 3.1.1.3 Прохождение теста
**Описание.** Пользователь имеет возможность пройти тест по переводу добавленных в словарь слов. 

| Функция | Требования | 
|:---|:---|
| Прохождение теста | Приложение должно предоставить пользователю возможность пройти тест по переводу слов |
| *В словаре недостаточно слов для составления теста*| *Приложение должно известить пользователя о том, что в словаре недостаточно слов для составления теста. Количество слов должно быть не менее 10* |
| Просмотр результатов | Приложение должно предоставить пользователю возможность посмотреть результаты прохождения теста. Должна быть выведена информация о том, насколько процентов успешно был пройден тест, а также показаны слова, в переводе которых была допущена ошибка |


<a name="restrictions_and_exclusions"/>

### 3.1.2 Ограничения и исключения
1. Приложение работает только на операционной системе Windows;
2. Приложение не работает без установленного .NET Framework версии 4.0 и выше;
3. Приложение не работает при отсутствии подключения к Интернету.

<a name="non-functional_requirements"/>

## 3.2 Нефункциональные требования

<a name="quality_attributes"/>

### 3.2.1 Атрибуты качества

<a name="requirements_for_ease_of_use"/>

#### 3.2.1.1 Требования к удобству использования
1. Доступ к основным функциям приложения не более чем за две операции;
2. Понятный графический интерфейс;
3. Слова в словаре сохраняются и выводятся при следующем запуске приложения;
4. Удаление нужного слова из словаря по желанию.

<a name="security_requirements"/>

#### 3.2.1.2 Требования к безопасности
Данное приложение является полностью безопасным, поскольку оно не собирает информацию о пользователе.

<a name="external_interfaces"/>

### 3.2.2 Внешние интерфейсы
Окна приложения удобны для использования пользователями с плохим зрением:
  * размер шрифта не менее 14пт;
  * функциональные элементы контрастны фону окна.

<a name="restrictions"/>

### 3.2.3 Ограничения
1. Приложение должно быть реализовано на платформе .NET Framework 4.0+ с использованием WPF;
2. Слова из словаря хранятся в реляционной базе данных (MS SQL);
3. Поиск слов осуществляется с использованием сервиса «Яндекс.Словарь».
